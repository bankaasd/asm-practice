# Практична робота №2: Взаємодія з операційною системою через `int 0x80`

## 1. Що таке `int 0x80`?

В архітектурі Linux x86 (32-біт) інструкція **`int 0x80`** (Interrupt 0x80 / Переривання 0x80) є програмним перериванням, яке використовується для переходу з простору користувача (User Space) до простору ядра (Kernel Space). 

Оскільки прикладні програми не мають прямого доступу до апаратного забезпечення (екран, диски, мережа), вони змушені звертатися до операційної системи із запитом на виконання цих операцій. `int 0x80` — це «ворота», через які програма викликає системні функції (System Calls).

### Як працює механізм системного виклику:
1. Номер системного виклику завантажується в регістр **`EAX`**.
2. Параметри (аргументи) виклику передаються через інші регістри загального призначення в чітко визначеному порядку: **`EBX`**, **`ECX`**, **`EDX`**, **`ESI`**, **`EDI`**.
3. Виконується інструкція `int 0x80`. Процесор призупиняє роботу програми, перемикає режим роботи на рівень ядра, виконує потрібну функцію ОС і повертає керування програмі.
4. Результат виконання (або код помилки) ядро повертає назад у регістр **`EAX`**.

## 2. Найбільш часто використовувані системні виклики Linux (x86)

Нижче наведено таблицю базових системних функцій, які використовуються під час низькорівневого програмування на Ассемблері для взаємодії з ОС.

| Номер виклику (EAX) | Назва функції | Опис | Параметри в регістрах |
| :---: | :--- | :--- | :--- |
| **1** | `sys_exit` | Коректне завершення роботи програми. | `EBX` — код повернення (0 для успіху). |
| **3** | `sys_read` | Читання даних із файлового дескриптора (наприклад, введення з клавіатури). | `EBX` — файловий дескриптор (0 для `stdin`).<br>`ECX` — покажчик на буфер у пам'яті.<br>`EDX` — максимальна кількість байт для читання. |
| **4** | `sys_write`| Запис даних у файловий дескриптор (наприклад, виведення тексту на екран). | `EBX` — файловий дескриптор (1 для `stdout`, 2 для `stderr`).<br>`ECX` — покажчик на рядок або дані.<br>`EDX` — кількість байт для виведення. |
| **5** | `sys_open` | Відкриття файлу або пристрою. | `EBX` — покажчик на ім'я файлу (рядок із нулем наприкінці).<br>`ECX` — режими доступу (напр. тільки читання, запис).<br>`EDX` — права доступу до файлу (permissions). |
| **6** | `sys_close`| Закриття раніше відкритого файлового дескриптора. | `EBX` — файловий дескриптор, який потрібно закрити. |

mov eax, 4         ; Номер системного виклику sys_write

mov ecx, prompt    ; Адреса нашого тексту в секції .data

mov eax, 1         ; Номер системного виклику sys_exit

int 0x80           ; Передаємо керування ядру ОС
